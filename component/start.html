Get Started

欢迎光临，你可以从这里开始无线开发之旅。这里面为您提供了页面脚手架，h5 ui组件，以及一些常用的js组件功能（建设中）。如果你是基于移动工作平台的开发方式，进入这里，如果你自己从头开发整个 页面，可以直接到这里.

一，框架开发方式

1,页面和静态文件目录规范

该种方式，移动工作平台提供了完整个h5页面开发框架，h5相关页面mobile_server/deploy/templates/mobile/vm下，如右图；业务应用之间用文件夹隔离

静态文件放在mobile_server/deploy/htdocs/h5下，业务应用之间也是用文件夹隔离，每个h5页面，会在静态文件夹下唯一对应一个js或css，静态文件也可以没有。

2,h5页面脚手架

由于框架已经将html的head部分抽出，这里应用页面可以专注写自己的逻辑。下面介绍2种典型页面结构，用户可以根据是否带滚动条选择其一。

2.1,普通页面框架标准 下面设置了h5的头部和尾部组件，在hybrid应用中，每打开一个页面，你可以调用native的头部和尾部，所以需要根据需要增删。

3,css开发

3.1,开发目录

如右图，业务样式，在css目录下，以文件夹隔离

3.2,开发方式

推荐sass方式，及可以类似java引用包的方式，引用我们的组件库，然后使用koala工具编译，即可看到效果。具体如下：

在css下建立业务文件夹，然后在业务文件夹中开始写页面的样式，文件命名与h5相同，后缀使用.scss，内容写法类似如右侧：

/* 引用base */
@import "../sass1.0/base/base.scss";
/* 引用组件 */
@import "../sass1.0/widget/ui-btn.scss";
@import "../sass1.0/widget/ui-form.scss";

/****************************************
 * page style 这里在这里覆写组件的样式，和页面自身特有的样式
 ****************************************/
.ui-layout-header .icon-back {
  margin-top: -44px;
}
.ui-paragraph {
  .ui-form-lbl {
    width: 66px;
  }
}
